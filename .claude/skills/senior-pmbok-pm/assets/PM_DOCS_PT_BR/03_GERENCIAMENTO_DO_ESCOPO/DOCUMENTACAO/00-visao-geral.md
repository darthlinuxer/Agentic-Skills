# Visão Geral — Gerenciamento do Escopo

Esta documentação orienta como definir, documentar, validar e controlar o escopo do projeto. O objetivo é garantir que somente o trabalho necessário seja realizado e que as entregas atendam às necessidades de negócio.

## Princípios

- **Clareza e completude**: o escopo deve ser compreensível para todas as partes interessadas.
- **Rastreabilidade**: requisitos, entregas, testes e aceites devem estar conectados.
- **Controle de mudanças**: alterações são esperadas e precisam de processo formal.
- **Validação contínua**: aceitação de entregas deve ocorrer ao longo do projeto.

## Como usar esta documentação

1. Preencha o `TEMPLATE.md` com as informações do seu projeto.
2. Use os links de documentação para entender cada seção e boas práticas.
3. Registre entradas no `INPUTS.md` para manter rastreabilidade.
