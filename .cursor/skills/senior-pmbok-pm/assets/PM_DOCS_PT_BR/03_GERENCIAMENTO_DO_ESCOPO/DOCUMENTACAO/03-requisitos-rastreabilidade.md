# 3. Requisitos e Rastreabilidade

Esta seção descreve como os requisitos serão documentados e rastreados até entregas, testes e aceites.

## 3.1 Documentação de Requisitos

**Objetivo**: garantir que todos os requisitos estejam completos, claros e verificáveis.

**Boas práticas**:
- Defina um repositório único (tooling) para requisitos.
- Use atributos padrão (ID, tipo, prioridade, fonte, critérios de aceitação etc.).
- Garanta consistência entre documentos (BRD, FRS, NFR, backlog, casos de uso).

## 3.2 Matriz de Rastreabilidade de Requisitos (RTM)

**Objetivo**: assegurar rastreabilidade bidirecional do requisito até a entrega.

**Boas práticas**:
- Mantenha a RTM atualizada ao longo do projeto.
- Garanta cobertura (requisitos implementados, testados e aceitos).
- Use a RTM para análise de impacto em mudanças.
